01
Fundamentos

Símbolos del UML

Antes de escribir una sola línea de código, debes descifrar qué significa cada símbolo en el diagrama. Son el lenguaje universal del diseño orientado a objetos.

Símbolo Nombre Significado en Java Ejemplo
+ Public Accesible desde cualquier clase public void metodo()
− Private Solo accesible desde la misma clase private String nombre
# Protected Accesible desde la clase y sus subclases protected int id
~ Package Accesible dentro del mismo paquete int valor
cursiva Abstract Método sin cuerpo / Clase no instanciable abstract void mover()
subrayado Static Pertenece a la clase, no a instancias static int contador
«interface» Interface Contrato: métodos sin implementación interface Volable
«abstract» Clase abstracta No se puede instanciar directamente abstract class Animal
«enum» Enumeración Conjunto fijo de constantes enum Color
ℹ
Tip: Los atributos siempre van en la sección del medio del rectángulo. Los métodos van en la sección de abajo. El nombre de la clase (y el estereotipo si lo hay) va arriba.
02
Metodología

Proceso paso a paso

Sigue este orden cada vez que recibas un diagrama UML. Saltarte pasos genera errores de compilación y diseño difíciles de detectar después.

1

Identifica todas las clases y su tipo

Lee el diagrama completo antes de escribir código. Separa mentalmente: clases normales, clases abstractas, interfaces y enums. Cada tipo tiene una declaración diferente en Java.

CLASE NORMAL
class Nombre { }
ABSTRACTA
abstract class Nombre { }
INTERFACE
interface Nombre { }
2

Crea un archivo .java por clase

En Java, cada clase pública debe estar en su propio archivo. El nombre del archivo debe coincidir exactamente con el nombre de la clase, incluyendo mayúsculas.

⚠
Si la clase se llama Animal, el archivo debe llamarse Animal.java. Un error de nombre = error de compilación.
3

Declara los atributos (sección del medio)

Cada línea de la sección de atributos del UML se convierte en un campo de la clase Java. La sintaxis es: visibilidad tipo nombre;

UML

- nombre : String
- edad : int
# peso : double
→

JAVA

private String nombre;
private int edad;
protected double peso;
4

Escribe el constructor

El constructor aparece como un método con el mismo nombre que la clase y sin tipo de retorno. Sus parámetros son los tipos entre paréntesis en el UML.

UML

+ Animal(String, int, double)
→

JAVA

public Animal(String nombre, int edad, double peso) {
  this.nombre = nombre;
  this.edad = edad;
  this.peso = peso;
}
5

Implementa los métodos (sección de abajo)

Cada método del UML indica: visibilidad, nombre, parámetros y tipo de retorno. Los métodos en cursiva son abstractos y no tienen cuerpo.

6

Añade herencia e interfaces

Las flechas del diagrama te dicen qué clases extienden a otras y qué interfaces implementan. extends para herencia, implements para interfaces.

7

Compila y prueba desde Main

Crea una clase Main.java con el método main. Instancia objetos y verifica que el comportamiento sea el esperado según el diagrama.

✦
Compila siempre en orden: primero las clases base (sin dependencias), luego las que dependen de ellas. Nunca al revés.
03
Estructura

Clases y sus partes

Una clase UML tiene tres zonas. Aprender a leer cada zona es la habilidad más básica y más importante.

Diagrama UML

«abstract» Animal
- nombre : String
- edad : int
# peso : double
+ Animal(String, int, double)
+ getNombre() : String
+ hacerSonido() : void
+ toString() : String

Código Java

Animal.javaJAVA
public abstract class Animal {

  private String nombre;
  private int edad;
  protected double peso;

  public Animal(String n, int e, double p){
    this.nombre = n;
    this.edad = e;
    this.peso = p;
  }

  public String getNombre(){ return nombre; }

  public abstract void hacerSonido();

  @Override
  public String toString(){
    return "Animal: " + nombre;
  }
}
04
Relaciones

Herencia · extends

La flecha con triángulo hueco apunta de la subclase hacia la superclase. En Java se traduce con extends. Recuerda: en Java solo se puede heredar de una clase.

→
Regla: La flecha siempre apunta hacia ARRIBA (hacia el padre). La clase que tiene la punta del triángulo es la clase PADRE. La clase en el origen de la flecha es la HIJA que hereda.

Subclase en UML

Leon
- tamanoMelena : double
+ Leon(String, int, double, double)
+ hacerSonido() : void
+ cazar() : String

Código Java

Leon.javaJAVA
public class Leon extends Animal {

  private double tamanoMelena;

  public Leon(String n, int e,
              double p, double tm){
    super(n, e, p); // llama al padre
    this.tamanoMelena = tm;
  }

  @Override
  public void hacerSonido(){
    System.out.println("¡ROAR!");
  }

  public String cazar(){
    return "El leon caza.";
  }
}
⚠
super() obligatorio: Si la clase padre tiene constructor con parámetros, la clase hija DEBE llamar a super(...) como primera línea del constructor. De lo contrario, error de compilación.
✅

Qué se hereda

Atributos y métodos public y protected. También los métodos implementados. El hijo puede sobrescribirlos con @Override.

❌

Qué NO se hereda

Atributos y métodos private. Tampoco los constructores (hay que llamarlos explícitamente con super()).

05
Contratos

Interfaces · implements

Las interfaces son contratos. En UML se representan con «interface» y la flecha es discontinua con triángulo hueco. En Java: implements. Una clase puede implementar varias.

Interface en UML

«interface» Volable
+ volar() : String
+ aterrizar() : void

Interface en Java

Volable.javaJAVA
public interface Volable {
  // Todos los métodos son
  // public abstract por defecto
  String volar();
  void aterrizar();
}
Aguila.java · implementa VolableJAVA
public class Aguila extends Ave implements Volable {

  @Override
  public String volar(){
    return "El águila vuela a " + alturaVuelo + "m";
  }

  @Override
  public void aterrizar(){
    System.out.println("Aterrizando...");
  }
}
✦
Diferencia clave: extends hereda código. implements firma un contrato: la clase promete tener todos los métodos de la interfaz. Si falta alguno → error de compilación.
06
Enumeraciones

Enums · «enum»

Un enum representa un conjunto fijo y conocido de valores. En UML aparece con el estereotipo «enum». Pueden tener atributos y métodos propios.

Enum en UML

«enum» Habitat
SELVA
SABANA
OCEANO
POLAR
- descripcion : String
+ getDescripcion() : String

Código Java

Habitat.javaJAVA
public enum Habitat {

  SELVA("Selva tropical"),
  SABANA("Sabana africana"),
  OCEANO("Océano abierto"),
  POLAR("Zona polar");

  private final String descripcion;

  Habitat(String desc){
    this.descripcion = desc;
  }

  public String getDescripcion(){
    return descripcion;
  }
}
Uso del enum en otra claseJAVA
// Usar el enum como tipo de atributo
private Habitat habitat;

// Asignar un valor
habitat = Habitat.SELVA;

// Comparar
if (habitat == Habitat.POLAR) { ... }

// En un switch
switch (habitat) {
  case SELVA: ... break;
  case OCEANO: ... break;
}
07
Flechas

Tipos de relaciones

Las flechas en UML tienen significados muy precisos. Confundirlas es uno de los errores más comunes al implementar.

HERENCIA

Línea sólida + triángulo hueco

La subclase hereda de la superclase. Se implementa con extends. La punta apunta al padre.

REALIZACIÓN

Línea discontinua + triángulo hueco

La clase implementa una interfaz. Se implementa con implements. La punta apunta a la interfaz.

ASOCIACIÓN

Línea sólida + flecha sólida

Una clase "conoce" a otra. Se implementa como un atributo del tipo de la otra clase.

COMPOSICIÓN

Línea + rombo relleno

La parte no puede existir sin el todo. Si el padre se destruye, el hijo también. Se implementa creando la instancia hija dentro del constructor del padre.

08
Verificación

Checklist antes de entregar

Haz clic en cada punto para marcarlo. Si alguno queda sin marcar, revísalo antes de compilar.

  • He creado un archivo .java por cada clase del diagrama
  • Las clases abstractas tienen abstract en la declaración
  • Los métodos abstractos no tienen cuerpo { }
  • Cada subclase llama a super() en su constructor
  • Los métodos heredados tienen @Override
  • Las interfaces solo tienen métodos sin implementación (o default)
  • El enum tiene sus constantes en MAYÚSCULAS
  • Los atributos private tienen getters y setters si son necesarios
  • He comprobado las multiplicidades (si hay *, uso List<Tipo>)
  • El toString() devuelve info útil del objeto
  • He probado la clase desde Main.java instanciando objetos
