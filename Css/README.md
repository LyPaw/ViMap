# CSS: Hojas de Estilo en Cascada y Diseno de Interfaces

## Capitulo 1: Fundamentos de las Hojas de Estilo en Cascada

CSS, Cascading Style Sheets, constituye el lenguaje formal que define la presentacion visual de los documentos HTML, estableciendo una separacion clara y fundamental entre el contenido, que se define en HTML, y la forma, que se define en CSS. Esta separacion permite modificar la apariencia completa de un sitio web alterando unicamente los archivos de estilo, sin necesidad de modificar la estructura HTML subyacente, lo que facilita el mantenimiento, la consistencia visual y la adaptacion a diferentes dispositivos y medios.

El nombre cascada proviene del algoritmo que resuelve conflictos entre reglas CSS cuando dos o mas reglas pretenden aplicar estilos diferentes al mismo elemento. El algoritmo considera tres criterios en orden decreciente de prioridad. El primer criterio es el origen del CSS: el estilo predeterminado del navegador tiene la menor prioridad, seguido del estilo declarado por el usuario en la configuracion del navegador, y el estilo del autor tiene la mayor prioridad. El segundo criterio es la especificidad del selector: los selectores mas especificos tienen prioridad sobre los mas genericos, garantizando que una regla dirigida a un elemento concreto prevalezca sobre una regla general. El tercer criterio es el orden de declaracion: a igual especificidad y origen, la ultima regla declarada en el codigo es la que se aplica, siguiendo el principio de que lo ultimo prevalece.

La especificidad se calcula mediante un sistema de cuatro valores numericos que representan diferentes niveles de especificidad. Los selectores de elemento y pseudoelementos, como p, h1 o ::before, suman una unidad al cuarto valor. Los selectores de clase, atributo y pseudoclases, como .clase, [atributo] o :hover, suman una unidad al tercer valor. Los selectores de ID, como #identificador, suman una unidad al segundo valor. Los estilos en linea, declarados directamente en el atributo style del elemento HTML, suman una unidad al primer valor y tienen la maxima especificidad.

La declaracion !important, anadida al final de una propiedad, anula cualquier otra regla independientemente de su origen, especificidad u orden de declaracion. Sin embargo, su uso debe evitarse rigurosamente porque rompe el flujo natural del algoritmo de cascada, dificulta la depuracion de estilos y hace que el codigo sea mas dificil de mantener y sobrescribir.

## Capitulo 2: Selectores Avanzados y Pseudoclases Estructurales

Los selectores de atributo permiten seleccionar elementos basandose en la presencia o el valor de sus atributos, proporcionando un control preciso sobre la seleccion de elementos. El selector con presencia selecciona todos los elementos que poseen el atributo especificado, independientemente de su valor. El selector con valor exacto selecciona elementos cuyo atributo tiene exactamente el valor especificado. El selector que comienza con selecciona elementos cuyo atributo comienza con el valor especificado, resultando util para seleccionar enlaces que apuntan a determinados dominios o protocolos. El selector que termina con selecciona elementos cuyo atributo finaliza con el valor especificado, permitiendo seleccionar archivos por su extension. El selector que contiene selecciona elementos cuyo atributo contiene la subcadena especificada en cualquier posicion de su valor.

Las pseudoclases estructurales seleccionan elementos segun su posicion en el arbol del documento, permitiendo aplicar estilos basados en la estructura del HTML sin necesidad de anadir clases adicionales. La pseudoclase first-child selecciona el primer elemento hijo de su elemento padre. La pseudoclase last-child selecciona el ultimo elemento hijo. La pseudoclase nth-child selecciona elementos hijos segun una formula que puede ser una palabra clave como odd para posiciones impares o even para pares, o una expresion aritmetica como 3n+1 para seleccionar cada tercer elemento empezando por el primero. La pseudoclase not selecciona todos los elementos que no coinciden con un selector dado, permitiendo aplicar estilos por exclusion.

Las pseudoclases de estado permiten aplicar estilos segun el estado del elemento o la interaccion del usuario. La pseudoclase hover se activa cuando el usuario situa el puntero sobre un elemento. La pseudoclase focus se activa cuando un elemento recibe el foco, tipicamente mediante navegacion por teclado o clic. La pseudoclase active se activa mientras el elemento esta siendo activado por el usuario, como durante un clic. La pseudoclase checked se aplica a elementos de formulario seleccionados, como casillas de verificacion o botones de opcion. Las pseudoclases disabled y required seleccionan elementos de formulario segun su estado de habilitacion o requisito.

## Capitulo 3: El Modelo de Caja y la Filosofia del Diseno Fluido

El modelo de caja constituye el fundamento teorico sobre el que se construye todo el diseno visual en CSS. Cada elemento HTML se representa como una caja rectangular compuesta por cuatro capas concentricas: el contenido, el relleno, el borde y el margen. El contenido es el area interior donde se muestra el texto, las imagenes o los elementos hijos. El relleno es el espacio transparente entre el contenido y el borde, que separa visualmente el contenido del borde del elemento. El borde rodea el relleno y el contenido, pudiendo personalizarse en estilo, grosor y color. El margen es el espacio transparente exterior que separa el elemento de los elementos adyacentes, evitando que las cajas se toquen entre si.

Por defecto, las propiedades width y height se aplican exclusivamente al area de contenido, de modo que el tamano total del elemento se calcula sumando el ancho del contenido, el relleno izquierdo y derecho, y el borde izquierdo y derecho. Esto puede complicar el calculo de las dimensiones, especialmente en disenos complejos con multiples elementos anidados. La propiedad box-sizing con valor border-box modifica este comportamiento para que width y height incluyan el contenido, el relleno y el borde, simplificando significativamente los calculos y reduciendo errores en la maquetacion.

La filosofia del diseno fluido propugna la utilizacion de unidades relativas en lugar de unidades absolutas para las dimensiones de los elementos. Los porcentajes permiten que los elementos se redimensionen proporcionalmente al tamano de su contenedor. La unidad de viewport, vw para el ancho y vh para la altura, permite dimensionar elementos en funcion del tamano de la ventana del navegador. La unidad rem, relativa al tamano de fuente del elemento raiz, facilita la creacion de disenos escalables donde todos los tamanos mantienen la proporcion. La unidad em, relativa al tamano de fuente del elemento actual, es util para componentes que deben escalar uniformemente.

## Capitulo 4: Flexbox: Diseno Unidimensional

Flexbox es un modelo de layout unidimensional que trabaja en una sola direccion, fila o columna, proporcionando un sistema potente y flexible para distribuir el espacio entre elementos y alinearlos dentro de un contenedor. El contenedor flex se activa mediante la propiedad display con el valor flex, estableciendo un contexto de formato flexible para todos sus hijos directos.

Las propiedades del contenedor controlan el comportamiento global del layout. La propiedad flex-direction establece la direccion principal del flujo, pudiendo ser fila, que dispone los elementos horizontalmente de izquierda a derecha, fila inversa, que los dispone horizontalmente de derecha a izquierda, columna, que los dispone verticalmente de arriba a abajo, o columna inversa, que los dispone verticalmente de abajo a arriba. La propiedad flex-wrap controla si los elementos deben ajustarse a multiples lineas cuando no caben en una sola linea, o si deben comprimirse para permanecer en una unica linea. La propiedad justify-content alinea los elementos a lo largo del eje principal, ofreciendo opciones como inicio, final, centro, espacio entre elementos, espacio alrededor de cada elemento, y espacio equitativo. La propiedad align-items alinea los elementos a lo largo del eje transversal, con opciones como estiramiento para llenar el contenedor, inicio, final, centro y linea base. La propiedad gap establece el espacio entre los elementos, tanto en el eje principal como en el transversal.

Los items flexibles tienen propiedades individuales que permiten controlar su comportamiento especifico. La propiedad flex-grow determina la capacidad del item para crecer y ocupar el espacio disponible cuando hay espacio sobrante en el contenedor, distribuyendose proporcionalmente segun los valores de crecimiento de cada item. La propiedad flex-shrink determina la capacidad del item para encogerse cuando no hay suficiente espacio para todos los items. La propiedad flex-basis establece el tamano inicial del item antes de distribuir el espacio restante. La propiedad align-items permite sobrescribir la alineacion establecida por el contenedor para un item especifico. La propiedad order permite alterar el orden de presentacion de los items independientemente de su orden en el codigo HTML.

## Capitulo 5: CSS Grid: Diseno Bidimensional

CSS Grid constituye un modelo de layout bidimensional que trabaja simultaneamente con filas y columnas, proporcionando un nivel de control sobre la disposicion de los elementos que supera las capacidades de cualquier tecnica anterior. El contenedor grid se activa mediante la propiedad display con el valor grid, estableciendo una cuadricula donde se colocan los elementos hijos.

Las propiedades del contenedor definen la estructura de la cuadricula. Las propiedades grid-template-columns y grid-template-rows definen el tamano y el numero de columnas y filas respectivamente, aceptando una lista de valores que pueden ser unidades absolutas como pixeles, unidades relativas como porcentajes, o la unidad especial fr que representa una fraccion del espacio disponible en el contenedor. La funcion repeat permite definir patrones de columnas o filas de forma concisa. Las funciones minmax y las palabras clave auto-fill y auto-fit permiten crear disenos adaptables que se ajustan automaticamente al espacio disponible sin necesidad de consultas de medios.

Los items grid se colocan automaticamente en las celdas disponibles siguiendo el orden del codigo HTML, pero pueden posicionarse explicitamente utilizando las propiedades grid-column y grid-row, que aceptan valores de linea de inicio y linea de fin, o la palabra clave span para indicar cuantas celdas debe ocupar el item. La propiedad grid-template-areas permite definir areas nombradas en el contenedor y asignar items a esas areas mediante la propiedad grid-area, facilitando la creacion de layouts complejos y semanticamente claros, como el patron clasico de pagina completa con cabecera, barra lateral, contenido principal, panel secundario y pie de pagina.

## Capitulo 6: Animaciones y Transiciones CSS

Las transiciones CSS permiten cambios suaves y graduales entre dos estados de un elemento, animando la variacion de una o mas propiedades cuando su valor cambia. Una transicion se define mediante la propiedad shorthand transition, que especifica la propiedad a animar, la duracion del efecto, la funcion de temporizacion que determina la aceleracion del cambio, y el retardo antes de iniciar la animacion. Las transiciones se activan automaticamente cuando cambia el valor de una propiedad, tipicamente en respuesta a interacciones del usuario como hover, focus o cambios de clase mediante JavaScript.

Las animaciones CSS, mas potentes y versatiles que las transiciones, permiten definir secuencias de cambios complejas con multiples puntos de control. La regla @keyframes define los puntos de la animacion utilizando porcentajes que representan el progreso de la animacion, o las palabras clave from y to como abreviaturas para el inicio y el final. Cada punto de control especifica los valores que deben tener las propiedades animadas en ese momento de la secuencia. La propiedad animation aplica la animacion a un elemento, especificando el nombre de la animacion, la duracion, la funcion de temporizacion, el numero de iteraciones, la direccion, si debe reproducirse en sentido normal, inverso o alternado, el estado inicial antes de que comience la animacion, y el estado final despues de que termine.

## Capitulo 7: Diseno Adaptable y Consultas de Medios

El diseno adaptable, o responsive design, constituye una filosofia de diseno y desarrollo web que busca que las paginas se visualicen correctamente en cualquier dispositivo, independientemente del tamano de su pantalla. En lugar de crear versiones separadas del sitio para diferentes dispositivos, el diseno adaptable utiliza una unica base de codigo que se adapta al contexto de visualizacion.

Las consultas de medios, o media queries, constituyen el mecanismo fundamental del diseno adaptable, permitiendo aplicar estilos condicionales basados en las caracteristicas del dispositivo o de la ventana del navegador. Las consultas pueden evaluar el ancho de la ventana, la altura, la orientacion, la resolucion, la relacion de aspecto, el tipo de dispositivo, y las preferencias del usuario como el esquema de color claro u oscuro o la reduccion de movimiento.

El enfoque mobile first consiste en escribir primero los estilos base para pantallas pequenas, sin consultas de medios, y luego anadir estilos adicionales para pantallas progresivamente mayores mediante consultas de medios con la condicion de ancho minimo. Este enfoque garantiza que el contenido basico sea accesible en cualquier dispositivo y que las funcionalidades avanzadas se anadan solo cuando el espacio de pantalla lo permita.

Las consultas de contenedor, o container queries, constituyen una evolucion reciente de las consultas de medios que permiten aplicar estilos basados en el tamano del contenedor padre de un elemento, en lugar del tamano de la ventana del navegador. Esto permite crear componentes verdaderamente reutilizables que se adaptan al espacio disponible en su contexto de uso, independientemente del tamano de la pantalla del dispositivo.

## Capitulo 8: Variables CSS y Personalizacion de Temas

Las variables CSS, tambien denominadas propiedades personalizadas, constituyen un mecanismo para definir valores reutilizables en todo el documento, facilitando la consistencia visual, la creacion de temas y el mantenimiento del codigo. A diferencia de las variables de los preprocesadores CSS, que se evaluan en tiempo de compilacion, las variables CSS se evaluan en tiempo de ejecucion y pueden ser modificadas dinamicamente mediante JavaScript.

Las variables CSS se declaran con un nombre que comienza con doble guion, y se definen tipicamente dentro del selector root para que esten disponibles globalmente en todo el documento. Para utilizar el valor de una variable, se emplea la funcion var, que acepta el nombre de la variable como primer argumento y, opcionalmente, un valor por defecto como segundo argumento que se utilizara si la variable no esta definida.

El ambito de las variables CSS sigue las reglas de la cascada, lo que significa que una variable definida en un selector especifico solo esta disponible para ese elemento y sus descendientes, permitiendo la creacion de temas locales y la sobrescritura selectiva de valores. Esta caracteristica, combinada con la capacidad de modificar variables mediante JavaScript, permite implementar sistemas de temas dinamicos, modos claro y oscuro, y personalizacion de la apariencia por parte del usuario sin necesidad de recargar la pagina ni de duplicar estilos.
